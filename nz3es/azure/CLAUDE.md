# Azure Infrastructure ‚Äî Claude Context

## Path Convention

```text
{org}/azure/{env}/{plane}/{project}/{region}/{component}/{instance}/
```

- Path components parsed by `root-azure.hcl`: `org`, `provider`, `environment`, `plane`, `project`, `region`, `component`
- Use `find_in_parent_folders("root-azure.hcl")` ‚Äî NOT `root.hcl`
- Use `get_repo_root()` for all module paths

## CAF Naming Convention

**Always follow Microsoft Cloud Adoption Framework naming.** Pattern: `{type}-{workload}-{env}-{project}-{region_short}`

### Resource Type Abbreviations

| Resource | Abbreviation | Example |
| -------- | ------------ | ------- |
| Resource group | `rg` | `rg-aks-stg-iac-01-ause` |
| Virtual network | `vnet` | `vnet-stg-iac-01-ause` |
| Subnet | `snet` | `snet-aks-nodes` |
| AKS cluster | `aks` | `aks-stg-iac-01-ause` |
| Managed identity | `id` | `id-aks-cluster-stg-iac-01-ause` |
| Key vault | `kv` | `kv-stg-iac-01-ause` |
| Storage account | `st` | `ststgiac01ause` (no hyphens, max 24 chars) |
| Container registry | `cr` | `cr-stg-iac-01-ause` |
| Private endpoint | `pep` | `pep-stg-iac-01-ause` |

**Abbreviation as prefix** (not suffix) ‚Äî resources sort by type in the Azure portal.

### Naming Formula in Deployment Units

```hcl
# Resource group (folder name = workload, no env/region in folder)
name = "rg-${basename(get_terragrunt_dir())}-${include.root.locals.environment}-${include.root.locals.project}-${include.root.locals.region_short}"

# AKS cluster
cluster_name = "aks-${basename(get_terragrunt_dir())}-${include.root.locals.region_short}"

# Managed identity (under global/ ‚Äî region from RG dependency, not root.hcl)
# Use include.root.locals.region_short_names (from root-azure.hcl) ‚Äî never redefine the map locally
name = "id-${local.identity_name}-${include.root.locals.environment}-${include.root.locals.project}-${lookup(include.root.locals.region_short_names, dependency.resource_group.outputs.location, dependency.resource_group.outputs.location)}"
```

**Never hardcode env (`stg`) or region (`ause`) in folder names** ‚Äî derive them from root locals.

## Module Patterns

### Upstream (tfr://)

```hcl

  source = "tfr:///Azure/aks/azurerm?version=9.1.0"
}
```

### Custom (local Terraform module)

```hcl
terraform {
  source = "${get_repo_root()}//modules/azure/resource-group"

```

### Module wrapper include

```hcl
include "aks" {
  path = "${get_repo_root()}/modules/azure/aks-cluster/terragrunt.hcl"

```

## Resource Group Design

One RG per lifecycle/team boundary (CAF recommendation):

| RG folder | Contains | Naming |
| --------- | -------- | ------ |
| `resource-group/network/` | VNet, subnets | `rg-network-{env}-{project}-{region}` |
| `resource-group/aks/` | AKS cluster, managed identities | `rg-aks-{env}-{project}-{region}` |

- Only the **state storage RG** (`nz3es-tf-state-rg`) is bootstrapped manually
- All deployment RGs are Terragrunt-managed
- AKS always creates a second hidden **node RG** (`MC_...`) automatically ‚Äî expected behaviour

## Dependency Pattern

```hcl
dependency "resource_group" {
  config_path = "${get_terragrunt_dir()}/../resource-group/network"
  mock_outputs = { name = "mock-rg", location = "australiaeast" }
  mock_outputs_allowed_terraform_commands = ["validate", "plan"]
}
```

Resources under `global/` path point to a regional RG ‚Äî Azure has no truly global resource scope unlike GCP.

## Workload Identity Pattern (Decoupled)

- **infra-as-code**: `managed-identity` unit creates `azurerm_user_assigned_identity` + `azurerm_federated_identity_credential`
- **k8s-as-code**: KSA annotated with `azure.workload.identity/client-id: <client_id>` via Helm
- Pod needs label `azure.workload.identity/use: "true"`
- AKS cluster must have `oidc_issuer_enabled = true` (set in module defaults)

## Provider & Auth

- `features {}` block is **mandatory** in HCL ‚Äî cannot be satisfied by env vars alone
- All credentials via env vars: `ARM_SUBSCRIPTION_ID`, `ARM_TENANT_ID`, `ARM_CLIENT_ID`, `ARM_CLIENT_SECRET`
- Provider block generated by `root-azure.hcl` ‚Äî do not add provider blocks in individual units

## State Backend

- Azure Storage Account: `nz3estfstate` in RG `nz3es-tf-state-rg`
- Container: `tfstate`
- Key: `infra-as-code/{path_relative_to_include()}/terraform.tfstate`

## Module Wrapper Rule

Module wrapper `inputs` must contain **only static literal values** (booleans, strings, numbers, empty maps/lists). If the wrapper `inputs` contain complex nested structures (e.g. `maintenance_window`, `node_pools = {}`), deep-merging with a deployment unit that has `dependency` references inside those same keys triggers `"There is no variable named dependency"` ‚Äî Terragrunt evaluates the merged result in the included file's context where `dependency` is not in scope.

**Safe:** wrapper `inputs` = static literals only (same pattern as `modules/gcp/gke-private-cluster/terragrunt.hcl`)
**Unsafe:** wrapper `inputs` contains nested maps/lists that the deployment unit also overrides with `dependency` references

**Fix:** remove `inputs` from the wrapper entirely; inline `terraform { source = "tfr://..." }` in the deployment unit with all inputs there.

## Common Gotchas

- Verify `Azure/aks/azurerm` and `Azure/vnet/azurerm` versions at registry before pinning
- AKS `private_cluster_enabled = true` requires access from within VNet (use `az aks command invoke` or jump VM)
- `rbac_aad_admin_group_object_ids` must be set ‚Äî `local_account_disabled = true` locks out password auth
- Networking uses **Azure CNI Overlay** (`network_plugin_mode = "overlay"`): pods get IPs from `pod_cidr` (virtual), only nodes use VNet IPs ‚Äî no pod subnet in VNet needed
- IP plan: VNet `10.1.0.0/22` | `snet-aks-nodes 10.1.0.0/24` | `snet-private-ep 10.1.1.0/27` | spare `10.1.2.0/24` + `10.1.3.0/24` | Service CIDR `172.16.0.0/16` | Pod CIDR `172.17.0.0/16` ‚Äî must not overlap
- Service CIDR and Pod CIDR are virtual ‚Äî never create VNet subnets for them
- `Azure/vnet/azurerm` requires `use_for_each = true` ‚Äî set in module wrapper default; omitting it causes index-based recreation when subnets are added/removed
- `Azure/aks/azurerm` requires `prefix` to be non-empty ‚Äî the module has an unconditional precondition on `log_analytics.tf` checking `coalesce(cluster_log_analytics_workspace_name, prefix)`; fails even when `oms_agent_enabled = false`; set `prefix = local.cluster_name` in the deployment unit
- `Azure/vnet/azurerm` VNet address space variable is `address_space` (NOT `vnet_address_space`) ‚Äî wrong name is silently ignored and defaults to `10.0.0.0/16`
- `Azure/vnet/azurerm` subnet output: use `vnet_subnets_name_id["<name>"]` for name‚Üíid map lookups; `vnet_subnets` is a plain list (numeric index only)
- `subnet_delegation` values must include `service_name` explicitly ‚Äî the module does `lookup(delegation.value, "service_name")` so omitting it causes `Invalid function argument` at apply time
- **Only `snet-aks-apiserver` should be delegated** to `Microsoft.ContainerService/managedClusters` ‚Äî delegating `snet-aks-nodes` causes `SubnetIsDelegated` 400 error when AKS tries to use it as the node subnet
- **`Azure/aks/azurerm v9.x` does NOT support API Server VNet Integration** ‚Äî the variable `api_server_vnet_integration_enabled` was added in v10+; setting `api_server_subnet_id` with v9.x causes `ApiserverSubnetConfigError` (400). Use Option C (IP whitelist only) with v9.x, or upgrade to v10+ for Option D
- Subnet names must match exactly when referenced in `vnet_subnets[...]` dependency outputs
- `global/` units lack a real region ‚Äî use `include.root.locals.region_short_names` with RG dependency location
- Storage account names: no hyphens, max 24 chars, globally unique across Azure
